freee のアカウントを作る 🏦

⏱ 約10分 🤝 準備ステップ

まずは伝票が起票される「箱」をご用意ください。クラウド会計freeeの無料アカウントを作成します。すでにfreeeのアカウントをお持ちの方はこのステップは飛ばしてOKです。

💡 これで何がラクになる？
freeeは「取引」を入れると、複式簿記の仕訳を自動で組み立ててくれるクラウド会計です。あとでAIがここに直接書き込めるようになります。
📋 手順
  1. freee公式サイト を開く
  2. 「無料で始める」からメールアドレスで登録
  3. 確認メールのリンクを開いて本登録
  4. 事業所名（屋号や会社名／個人なら自分の名前でOK）を入力
  5. ログイン後、ダッシュボードが表示されれば完了です！
🔒 安全メモ
パスワードの使い回しはお控えください。ログイン情報やトークンが映る画面は📸スクリーンショット・画面共有をしないようご注意ください。
🛠 迷ったとき・トラブル
  • 確認メールが来ない → 迷惑メールフォルダをご確認ください。数分待っても来ない場合は再送してください。
  • 事業所はあとで変えられる？ → はい、設定からいつでも変更可能です。気軽に進めていただいて大丈夫です。
  • 有料プランに入る必要は？ → freeeは新規登録から30日間のお試し期間であれば、口座連携・freee MCP（API連携）を無料でご利用いただけます。ただしお試し期間が終了し有料プランをご契約いただかないと、口座連携・API連携は自動的に解除・停止されます。今日の体験のみであれば問題ありませんが、実務で継続利用される場合は有料プランのご契約が必須です。

銀行口座・クレジットカードを連携する 🏧

⏱ 約10分 🤝 準備ステップ

freeeに口座やカードをつなぐと、明細が自動で取り込まれます。これが「AIが自動で起票する」体験の元になります。すでに口座連携が済んでいる方はこのステップは飛ばしてOKです。

💡 これで何がラクになる？
口座・カードを連携すると、入出金明細が自動でfreeeに届くようになります。あとはAIがその明細を見て「これは消耗品費っぽいですね、登録しますか？」と提案し、起票するという流れが作れます。
📋 手順
  1. freeeにログインし、左メニューの「口座」（または「自動で経理」）を開く
  2. 「新しく口座を登録」→ 連携したい銀行・クレジットカード会社を検索
  3. 各金融機関のネットバンキングID・パスワードを入力してログイン許可
  4. 同期が始まったら完了です。初回は明細の取り込みに数分〜数十分かかることがあります
🔒 安全メモ
ネットバンキングのID・パスワードはfreee社の正規連携画面以外には絶対に入力しないでください。連携作業中の画面は📸スクリーンショット・画面共有をしないようご注意ください。
🛠 迷ったとき・トラブル
  • 連携したい銀行が見つからない → 銀行名・支店名で再検索してください。それでも出ない場合はCSVでの明細取込に切り替え可能です（当日サポートいたします）。
  • 明細がなかなか出てこない → 初回同期は時間がかかることがあります。10〜20分待っても出ない場合は「今すぐ同期」を再実行してください。
  • 連携自体に料金はかかる？ → 新規登録から30日間のお試し期間中は無料で口座連携をご利用いただけます。お試し期間後も連携を続けるには有料プランのご契約が必要です（未契約のままだと連携が解除されます）。
  • すでに連携済みでテスト用の明細がない → ステップ④に「テスト取引を手動で起票してもらう」コピペ文もご用意していますので、そちらをお使いください。

Claude Code をインストールする 💻

⏱ 約15分 ⚡ セットアップ

次は、AIに作業をお願いするための道具「Claude Code」をご自身のPCに入れます。インストール手順は事前に共有しているUnikoukokunさんの課題ページにまとまっていますので、基本的にはそちらを見ながら進めてください。

💡 これで何がラクになる？
Claude Codeは、AIに自然な言葉で指示できるデスクトップアプリです。「この請求書を登録して」のような依頼を、そのまま実行に変えてくれる相棒です。ターミナル（黒い画面）は不要で、デスクトップアプリだけで完結します。
📋 進め方
  1. こちらのページを開く → Unikoukokunさんの課題ページ（課題1-1）
  2. 「課題1-1」の手順に沿って、Mac / Windowsどちらかご自身のPCに合う方でインストール（この手順の中で、Claude Codeのインストール〜デスクトップアプリの起動まで一通り完了します）
  3. アカウント作成は「Googleで続ける」から、ふだん使っているGoogleアカウントを選ぶだけでOK
  4. Claude を課金（Proプラン・月額制／約22ドル＝3,300円前後）する。課金しないとClaude Codeは動きませんので、ここは必須です
  5. （Gitが必要と案内された場合のみ）Mac は GitHub Desktop、Windows は git-scm.com からGUIインストール。インストール後はPCを再起動するかアプリを開き直す

Claude Codeアプリを開いて「こんにちは」と話しかけてみて、返事が来れば成功！

📝 Claude Codeが作った資料はどこで見る？
Claude Codeにお願いすると、議事録やメモなどをMarkdown（.md）ファイルとして作ってくれることがあります。Wordなど別のアプリを開く必要はなく、Claude Codeデスクトップアプリの中のファイル一覧・プレビュー画面でそのまま読めますので、この先の課題でも「作ってもらったファイルはアプリ内で見る」が基本と覚えておいてください。
🔒 安全メモ
クレジットカード情報の入力画面・ログイン画面は📸スクリーンショット・画面共有をしないようご注意ください。
🛠 迷ったとき・トラブル
  • 課金しないと使える？ → 使えません。無料プランのままではClaude Codeは動きません。月額のProプラン（年額ではなく月額を選択）にご加入ください。
  • Googleアカウントがない → メールアドレスでの新規登録も可能です。当日サポートいたします。
  • Gitを案内されたけどよく分からない → アプリの案内に従ってGUIインストールするだけでOKです。コマンド入力は不要です。
  • インストールしたのに反応しない → アプリを完全に閉じて開き直してください。それでも直らない場合はPCを再起動してください。
  • 作ってもらったmdファイルが見当たらない → Claude Codeアプリのファイル一覧（サイドバー）をご確認ください。「〇〇というファイル名で作った」とAIが答えた名前で探すと見つかりやすいです。

freee MCPサーバーを接続して、自動起票を体験する 🔌

⏱ 約20分 🚀 本番・体験

いよいよ本番です！ Claude Code と freee を「MCP」でつなぐと、AIがあなたの代わりにfreeeへ伝票を起票できるようになります。ターミナル（黒い画面）は使いません。Claude Codeデスクトップアプリのチャットに文章を貼るだけで、接続作業はClaude自身が進めてくれます。

💡 MCPってなに？
MCP（Model Context Protocol）は、AIと外部サービスをつなぐ“共通プラグ”。これを挿すと、Claude Codeが「freeeを操作する手」を手に入れるイメージです。
📋 手順①：freee MCPプラグインを入れる（デスクトップアプリに貼るだけ）
  1. Claude Codeデスクトップアプリを開き、新しいチャットを開始する
  2. 下のコピペ文をそのままチャット欄に貼り付けて送信する（freee公式が配布している正規のプラグインを、Claude自身がインストールしてくれます）
  3. 途中で「このコマンドを実行してもよいですか？」のような許可の確認が表示されたら、内容を読んで「許可」を選ぶ
  4. Claudeから「インストールが完了しました」の報告が来たら、アプリを一度完全に終了して、開き直す（開き直すことで、追加したプラグインが読み込まれます）
📋 ① デスクトップアプリに貼るだけ：freee MCPプラグインのインストール
freee会計とClaude Codeを連携したいので、freee公式のMCPプラグインをこのPCにインストールしてください。

やることは、次の2つのコマンドを順番に実行するだけです。
1. claude plugin marketplace add freee/freee-mcp
2. claude plugin install freee-mcp@freee-mcp-marketplace

終わったら、うまくいったかどうか結果を報告してください。
エラーが出た場合は、原因と私がやるべきことを分かりやすく教えてください。
📋 手順②：freeeと認証する（これも貼るだけ）
  1. 開き直したClaude Codeデスクトップアプリで、下のコピペ文を送信する
  2. ブラウザでfreeeのログイン画面が自動で開く → freeeにログインして「許可する」をクリック
  3. 連携したい事業所を選択する（ステップ①「freeeアカウント作成」で作った事業所を選ぶ）。複数の事業所が並ぶ方（税理士・複数社経営の方）はここが最重要ポイントです。顧問先など本番の事業所を選ばないよう、事業所名をよく確認してから選んでください
  4. Claudeが「接続できました」と事業所名を報告してくれたら成功です！🎉

※ 権限はログイン中のfreeeユーザーと同じ範囲に限定されます。

📋 ② デスクトップアプリに貼るだけ：freeeと認証する
freee MCPの認証（freeeとの接続）を進めてください。
ブラウザが開いたら、私がfreeeにログインして「許可する」を押します。
接続が完了したら、どの事業所につながったかを確認して教えてください。
🏃 メイン体験：口座連携した明細から、AIに自動で起票してもらう
最初に、いま接続しているfreeeの事業所名を教えて。
私が事業所名を確認して「OK」と言うまで、先に進まないで。

OKと言ったら、freeeに同期されている未処理の取引明細（自動で経理）を1件見せて。
内容を踏まえて、勘定科目の候補を提案してね。
もしこの明細が、既存の未決済取引（請求書・給与などの未払い）の支払にあたる場合は、
新規の取引ではなく、その未決済取引への決済登録として処理してね。
私がOKと言ったら、その内容でfreeeに登録して。

Claudeが明細を確認 → 勘定科目を提案 → あなたが「OK」と返す → freee側に取引が自動で起票される。これが今日のメインゴールです✨ freeeの「取引」一覧を開いて、登録されているか確認してみましょう！

🙋 ステップ②（口座連携）を飛ばした／同期明細がまだない方はこちら

口座連携がまだ、または明細の取り込みが間に合っていない場合は、代わりにこちらをコピペしてください。テスト取引で同じ流れを体験できます。

📋 テスト用：AIに伝票を起票してもらう
最初に、いま接続しているfreeeの事業所名を教えて。私が事業所名を確認して「OK」と言ったら、
freeeに、本日付で「消耗品費 1,100円（税込）／現金」の取引を1件、テストで起票して。
登録前に内容を見せて、OKしたら登録してね。
🔒 安全メモ
  • 複数の事業所につながる可能性がある方（税理士・複数社経営の方）は、起票前に必ず「いま接続している事業所名」を確認する。顧問先など別の事業所への誤登録が、今日いちばん防ぎたい事故です（上のコピペ文には確認の一文を入れてあります）。
  • 必ず「登録前に内容を見せて」と添えて、確認してから登録させる癖をつける。
  • 最初は少額・少件数で試すのが安心。
  • 登録した後も、freeeの「自動で経理」には同じ明細が未処理のまま残ります（明細を消込済みにするAPIがfreeeから提供されていないため）。二重登録を防ぐため、登録後にfreee画面で該当明細を「無視」にするか、登録済みの取引との紐づけを手動で行ってください。
  • 認証画面・トークンが映る画面は📸撮らない・共有しない。
🛠 迷ったとき・トラブル
  • 🟥 認証はOKなのに「事業所が見つからない」 → freeeにログインしている事業所と、認証時に選んだ事業所が同じか確認。
  • 🟥 起票したのにfreeeに出ない → 取引一覧の日付フィルタを確認。今日の日付に絞って再表示。
  • 🟥 登録したのに「自動で経理」の明細が未処理のまま残る → これは仕様です。明細側の消込を行うAPIがfreeeから提供されていないため、freee画面で該当明細を「無視」にするか、登録済みの取引と手動で紐づけてください。放置すると二重登録の原因になります。
  • 🟥 ①で「gitが見つかりません」のようなエラーが出る → プラグインの取得にGitというツールが必要です。ステップ③の手順5と同じ方法（Mac＝GitHub Desktop／Windows＝git-scm.comからGUIインストール）でGitを入れて、PCを再起動してから①をやり直してください。
  • 🟥 ②を送っても「freeeのツールが見つからない」と言われる → ①のあとのアプリ再起動がまだの可能性が高いです。アプリを完全に終了して開き直してから（タスクトレイ・Dockに残っていないかも確認）、もう一度②を送信してください。
  • 🟥 違う事業所に登録してしまった → あわてなくて大丈夫です。まずその事業所のfreee「取引」一覧から該当の取引を削除。そのあとClaudeに「接続する事業所を切り替えて」と頼んで、正しい事業所につなぎ直してください。
  • テストで起票した取引を消したい → freeeの「取引」一覧から該当の取引を開いて削除できます。Claudeに「さっきテスト起票した取引を削除して。削除前に内容を見せて」と頼んでもOKです。
  • 🟥 Claudeが「claudeコマンドが見つからない」と言う → Claude Codeアプリが古い可能性があります。アプリを最新版に更新して開き直し、もう一度①のコピペ文を送信してください。
  • 🟥「/plugin ...」だけを貼ったら反応しない → 「/」で始まるコマンドの断片だけを貼ると、チャットでは動きません。必ず上のコピペ文をまるごと貼って、Claudeに実行をお願いしてください。
  • プラグインが見つからない／インストールに失敗する → チャットで「freee-mcpのマーケットプレイスが追加されているか確認してください」と送信して状態を確認し、もう一度①からやり直してください。
  • 認証のブラウザ画面が開かない → チャットで「freeeの認証をもう一度やり直してください」と送信。それでも開かない場合はアプリを再起動してから再送してください。
  • 権限エラー → freee側でアプリ連携を「許可」したか再確認。チャットで「freee MCPの接続状態を確認してください」と送信して調べてもらうのもアリです。
  • 同期明細が1件も出てこない → ステップ②の口座連携がまだの可能性があります。上の「テスト用」コピペで体験を進めてOKです。

時間がある人向け：AI事務所テンプレート一式を入れる

⏱ 約1〜1.5時間 🌟 任意（チェック対象外）

ここまでで「自動起票デビュー」は完了です。時間に余裕がある方だけ、ここから先に進んでみてください。畠山から配布している「AI事務所テンプレート一式」を使うと、Claude Codeにあなた専用のキャラ・事務所ルール・安全ロック・記憶機能を一気に持たせられます。

🪟 先に確認：Windowsの方は「Git」と「Python」が必要です
テンプレートのうち「②セキュリティ」と「④Hooks」は、Gitに付属するGit BashとPythonという道具を使います。ステップ③④でGitをまだ入れていない方は、先に git-scm.com からGitを、python.org からPythonをインストールしてください（どちらも画面に沿って進めるだけでOK。迷ったらClaudeに「GitとPythonのインストールを案内して」と頼めば道案内してくれます）。Macの方は標準搭載なので、追加インストールは基本的に不要です。
💡 何が手に入る？
  • ① md設定：AIの名前・性格、事務所のルール（CLAUDE.md）
  • ② セキュリティ：.env誤読み取り・外部送信・rm -rf事故を防ぐロック
  • ③ メモリ：「前にも言いましたよね」を実現する記憶の仕組み
  • ④ Hooks：長い会話・要約後も前提を忘れない仕掛け
📖 詳細解説はこちら（任意）
📋 手順
  1. 下のリンクからテンプレート一式（フォルダ）をダウンロードし、デスクトップなど分かりやすい場所に保存 → AI事務所テンプレート一式（Googleドライブ）
  2. Claude Codeで、保存したフォルダを開く
  3. 下のコピペ文を会話画面に貼って送信（{{ }}の部分はご自身の情報に書き換えてください）↓
📋 Claude Codeに丸ごと渡すプロンプト
このフォルダには、Claude Codeのセットアップ一式（4点セット）が入っています。
セットアップ/README.md を読んで、書かれている順番（①md→②セキュリティ→③メモリ→④Hooks）の通りに、
私のPCへの設置を進めてください。

私は{{税理士／会計士／社労士／行政書士／弁護士／その他}}で、
事務所名は「{{あなたの事務所名}}」です。

進め方のルール：
- 各フォルダのREADMEに書いてある手順に沿って進めてください
- カスタマイズに必要な情報（名前・呼び方・キャラの性格など）は、その都度1問ずつ私に聞いてください
- settings.json など既存の設定ファイルを変更する前に、必ず日付つきのバックアップコピー（例: settings.json.bak_YYYYMMDD）を作ってください
- 既存ファイルの上書き・バックアップなど、確認が必要な場面では必ず先に私に確認してください

あとはClaude Codeが質問してくるので、答えながら進めるだけです。「コードが分からないから無理」と思う必要はありません。一気に4つ進めるのが不安なら「まずは①だけ進めてください」と区切ってお願いしてもOKです。
